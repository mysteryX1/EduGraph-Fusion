================================================================================
模块详情
================================================================================

【Models 层 - 数据定义】

1. KGNode - 知识图谱节点
2. KGEdge - 知识图谱边
3. Citation - 引用信息
4. Chapter - 章节数据
5. Textbook - 教材信息
6. KGExtraction - 知识提取结果
7. MergeDecision - 合并决策
8. RagAnswer - RAG 答案
9. ParseResult - 解析结果
10. TextbookMetadata - 教材元数据
11. Stats - 统计信息

【Services 层 - 业务逻辑】

1. ChapterParser - 章节识别
   • 支持 5 种章节标题格式
   • 正则表达式匹配

2. PDFParser - PDF 解析（逐页处理）
   • PyMuPDF 库支持
   • 内存高效
   • 自动章节识别

3. MarkdownParser - Markdown 解析
   • 按 # 标题分割
   • 保持原始格式

4. TextParser - 纯文本解析
   • 按固定字数分割
   • 自动生成伪章节

5. ParserFactory - 解析器工厂
   • 工厂设计模式
   • 灵活的扩展性

6. FileStorage - 文件存储
   • 文件上传和保存
   • 元数据管理
   • JSON 序列化
   • 完整的 CRUD 操作

7. StatsReporter - 统计报告
   • 全局统计
   • 详细统计
   • 教材级别统计

【Routers 层 - API 端点】

1. POST /api/upload
   • 接收文件上传
   • 支持 PDF、MD、TXT
   • 文件验证
   • 大小限制（100MB）

2. POST /api/parse/{textbook_id}
   • 解析教材文件
   • 自动选择解析器
   • 保存解析结果
   • 支持自定义 chunk_size

3. GET /api/textbooks
   • 获取教材列表
   • 分页支持
   • 排序和过滤

4. GET /api/textbooks/{textbook_id}
   • 获取教材详情
   • 包含章节列表
   • 完整的元数据

5. GET /api/stats
   • 全局统计信息
   • 文件类型分布
   • 上传时间分布
   • 排名数据

6. GET /api/textbooks/{textbook_id}/stats
   • 教材级别统计
   • 章节分析
   • 字数统计
