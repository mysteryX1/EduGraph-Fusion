## 数据流设计

### 上传和解析流程
```
用户上传文件
    ↓
前端校验 (类型、大小)
    ↓
POST /api/upload
    ↓
后端保存临时文件
    ↓
生成 textbook_id
    ↓
返回上传成功和 ID
    ↓
用户点击解析
    ↓
POST /api/parse/{textbook_id}
    ↓
后端调用对应 Parser
    ↓
提取内容和章节信息
    ↓
保存解析结果到 backend/data/processed/
    ↓
返回解析统计信息
```

### 知识图谱构建流程
```
获取已解析教材
    ↓
提取知识概念
    ↓
建立概念关系
    ↓
生成图结构 (nodes + links)
    ↓
缓存或持久化
    ↓
前端 ECharts 渲染
    ↓
支持交互 (缩放、拖拽、点击)
```

### RAG 检索流程
```
POST /api/rag/index
    ↓
加载已解析的教材数据
    ↓
文本分块和向量化
    ↓
建立向量索引 (FAISS)
    ↓
保存到 backend/data/vector_index/
    ↓
---
用户输入问题
    ↓
POST /api/rag/query
    ↓
问题向量化
    ↓
在索引中搜索相似文本
    ↓
返回 top-k 结果
    ↓
（可选）调用 LLM 生成答案
    ↓
返回答案 + 引用来源
```

## API 接口设计

### 统一响应格式

#### 成功响应
```json
{
  "status": "success",
  "message": "操作成功",
  "data": {
    // 具体数据
  }
}
```

#### 错误响应
```json
{
  "status": "error",
  "message": "错误描述",
  "path": "/api/xxx"
}
```

### 核心接口

#### 1. 文件上传
- **POST** `/api/upload`
- **请求**: FormData (file)
- **返回**: { textbook_id, filename, file_type, file_size }

#### 2. 文件解析
- **POST** `/api/parse/{textbook_id}`
- **参数**: chunk_size (可选)
- **返回**: { chapter_count, total_words, chapters[] }

#### 3. 教材列表
- **GET** `/api/textbooks`
- **参数**: limit, offset
- **返回**: { total, textbooks[] }

#### 4. 知识图谱构建
- **POST** `/api/kg/build`
- **参数**: textbook_ids[]
- **返回**: { nodes[], links[] }

#### 5. 知识图谱获取
- **GET** `/api/kg`
- **返回**: { nodes[], links[] }

#### 6. RAG 索引
- **POST** `/api/rag/index`
- **返回**: { indexed, chunk_count, textbook_count }

#### 7. RAG 查询
- **POST** `/api/rag/query`
- **请求**: { question, top_k }
- **返回**: { answer, citations[], source_chunks[] }

#### 8. 反馈提交
- **POST** `/api/feedback`
- **请求**: { type, content, related_node }
- **返回**: { feedback_id }

#### 9. 报告生成
- **POST** `/api/report/generate`
- **返回**: { report_id, status }

## 数据模型

### Textbook
```python
{
  "id": "textbook_xxx",
  "filename": "filename.pdf",
  "title": "教材名称",

  "upload_time": "2024-01-01T00:00:00Z",
```

### Chapter
```python
{
  "id": "chapter_xxx",
  "chapter_num": 1,
  "title": "第一章 绪论",
  "start_page": 1,
  "end_page": 30,
```

```python
{
  "id": "node_1",

```

### GraphLink
```python
{
  "source": "node_1",
  "target": "node_2",
  "value": 10,  # 关联强度

}
```

## 前端-后端通信

### 代理配置
在 vite.config.js 中配置代理，所有 `/api/*` 请求转发到后端：
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8000',
    changeOrigin: true,
  }
}
```

### API 超时设置
- 默认超时：30 秒
- 上传文件：60 秒
- 可根据需要调整

### 错误处理策略
1. 网络错误 → Mock 数据 fallback
2. 服务错误 → 显示友好的错误提示
3. 参数错误 → 表单验证和提示
4. 超时 → 重试或降级处理
