# EduGraph Fusion API 接口文档

## 基本信息

- **基础 URL**: `http://localhost:8000/api`
- **响应格式**: JSON
- **字符编码**: UTF-8
- **超时时间**: 30 秒（上传文件 60 秒）

## 统一响应格式

### 成功响应
```json
{
  "status": "success",
  "message": "操作成功",
  "data": {
    // 具体业务数据
  }
}
```

### 错误响应
```json
{
  "status": "error",
  "message": "错误描述信息",
  "path": "/api/xxx"
}
```

## 错误代码

| 代码 | 含义 | 说明 |
|------|------|------|
| 400 | Bad Request | 请求参数错误 |
| 404 | Not Found | 资源不存在 |
| 500 | Server Error | 服务器内部错误 |

## 1. 文件管理接口

### 1.1 上传文件

**端点**: `POST /upload`

**功能**: 上传教材文件（PDF、Markdown、纯文本）

**请求**:
```http
POST /api/upload HTTP/1.1
Content-Type: multipart/form-data

file: <file content>
```

**支持的文件格式**:
- `.pdf` - PDF 文件
- `.md`, `.markdown` - Markdown 文件
- `.txt` - 纯文本文件
- 最大大小: 100 MB

**成功响应** (200):
```json
{
  "status": "success",
  "message": "File uploaded successfully",
  "data": {
    "textbook_id": "textbook_abc123def456",
    "filename": "高中数学.pdf",
    "file_type": "pdf",
    "file_size": 5242880
  }
}
```

**错误响应** (400):
```json
{
  "status": "error",
  "message": "Unsupported file type. Allowed: .pdf, .md, .markdown, .txt"
}
```

**示例**:
```bash
curl -X POST -F "file=@高中数学.pdf" http://localhost:8000/api/upload
```

---

### 1.2 解析文件

**端点**: `POST /parse/{textbook_id}`

**功能**: 解析上传的教材文件，提取章节和内容

**路径参数**:
- `textbook_id` (string, required): 教材 ID

**查询参数**:
- `chunk_size` (integer, optional): 固定分块大小，默认 1000

**成功响应** (200):
```json
{
  "status": "success",
  "message": "Parsing completed successfully",
  "data": {
    "textbook_id": "textbook_abc123def456",
    "chapter_count": 12,
    "total_words": 150000,
    "chapters": [
      {
        "id": "ch_1",
        "chapter_num": 1,
        "title": "第一章 绪论",
        "start_page": 1,
        "end_page": 30,
        "word_count": 12500
      }
    ]
  }
}
```

**示例**:
```bash
curl -X POST http://localhost:8000/api/parse/textbook_abc123def456?chunk_size=1500
```

## 2. 教材管理接口

### 2.1 获取教材列表

**端点**: `GET /textbooks`

**功能**: 获取所有已上传的教材列表

**查询参数**:
- `limit` (integer, optional): 返回数量，默认 100，最大 1000

**成功响应** (200):

```json
{

    "total": 5,
    "limit": 100,
    "offset": 0,
    "textbooks": [
      {
        "id": "textbook_abc123def456",

        "chapter_count": 12,
        "total_words": 150000,
        "total_pages": 450

    ]
  }
}
```

**示例**:

```bash
curl http://localhost:8000/api/textbooks?limit=10&offset=0
```

```json
  "data": {
```

```json
    "answer": "函数是一种将一个集合中的每个元素映射到另一个集合中的唯一元素的关系...",

      "高中数学 - 第2章 - 第23-30页",
      "高中数学 - 第3章 - 第45-60页"
    ],
    "source_chunks": [
      {
        "content": "函数定义的详细内容...",
        "source": "高中数学",
        "chapter": "第2章",
        "page": "23-30"
      }
    ]
  }
}
```

## 5. 反馈接口

### 5.1 提交反馈

**端点**: `POST /feedback`

**功能**: 收集用户反馈

**请求体**:
```json
{

  "content": "知识图谱中某个节点显示错误",
```

**参数说明**:
- `type` (string, required): 反馈类型，可选值: `bug`, `suggestion`, `question`, `other`
- `content` (string, required): 反馈内容
- `related_node` (string, optional): 相关的知识点

## 常见问题与解决

### Q: 如何处理大文件上传？
A: 前端建议使用分片上传，后端已支持最大 100MB 文件。可考虑：
1. 拆分成更小的文件后分别上传
2. 使用浏览器的分片 API
3. 压缩文件后再上传

### Q: RAG 索引需要多长时间？
A: 取决于教材数量和大小：
- 小规模（<10 本）: ~10 秒
- 中规模（10-50 本）: ~1-2 分钟
- 大规模（>50 本）: 需要后台运行

### Q: 图谱合并的相似度阈值是多少？
A: 默认 0.85，可根据需要调整。相似度基于：
- 名称相似性
- 定义相似性
- 出现上下文相似性

### Q: 能否导出知识图谱？
A: 当前支持 JSON 格式导出。可通过调用 GET /kg 获取数据后自行处理。
