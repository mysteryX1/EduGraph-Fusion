================================================================================
前端集成信息
================================================================================

核心接口 (8 个)：

RAG 模块:
  1. POST /api/rag/index
     └─ 初始化索引，返回 {indexed, chunk_count, textbook_count}

  2. POST /api/rag/query
     └─ 查询知识库，返回 {answer, citations, source_chunks}

  3. GET /api/rag/status
     └─ 获取索引状态，返回 {indexed, chunk_count, textbook_count}

反馈模块:
  4. POST /api/feedback
     └─ 提交反馈，支持 instruction/feedback/text/content，返回 {action, summary, stats}

  5. GET /api/feedback/summary
     └─ 获取反馈摘要，返回 {total_decisions, keep_count, delete_count, split_count, merge_count, kg_nodes, kg_edges}

报告模块:
  6. POST /api/report/generate
     └─ 生成报告，返回 {report_file, report_path, generated_at, summary}

  7. GET /api/report/latest
     └─ 获取报告正文，返回 {report_file, report_path, modified_at, content}

  8. GET /api/report/summary
     └─ 获取报告摘要，返回 {textbooks, knowledge_graph, decisions}

前端示例代码: 见 FRONTEND_INTEGRATION_GUIDE.md

已知限制和注意事项
================================================================================

1. LLM 集成
   - 当前阶段不强制 LLM 可用
   - 如果未配置 LITELLM_PROVIDER/OPENAI_API_KEY/QWEN_API_KEY，自动跳过
   - Fallback: 直接返回上下文中最相关的内容

2. 反馈字段兼容
   - 后端接受 instruction/feedback/text/content 四个字段
   - 优先级: instruction > feedback > text > content
   - 前端可使用任意字段，后端会自动识别

3. merge_decisions.json 格式
   - 后端自动兼容 list (来自模块 A) 和 dict (来自模块 B) 格式
   - 内部统一转换为 dict，保存时也是 dict 格式

4. 报告正文
   - content 字段包含完整 Markdown 文本
   - 前端需要使用 markdown 库 (markdown-it, marked 等) 渲染为 HTML
