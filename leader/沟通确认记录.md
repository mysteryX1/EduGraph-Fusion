# EduGraph Fusion 沟通确认记录

> 本文件用于记录需要与项目 leader 确认、对齐和后续追踪的事项。后续凡是需要用户确认的关键内容，统一补充到 `leader/` 目录下。

## 2026-05-10 当前确认

### 项目技术栈

后端当前采用：

- Python + FastAPI
- Uvicorn 启动服务
- Pydantic 数据模型
- PyMuPDF 解析 PDF
- 本地 JSON 持久化，主要目录包括 `backend/data/metadata` 和 `backend/data/processed`
- RAG 方向预留或已有实现：`sentence-transformers`、`FAISS`，不可用时使用 `scikit-learn` / TF-IDF fallback
- LLM 调用方向预留：LiteLLM / OpenAI-compatible API / ModelScope 模型配置

前端当前采用：

- React 18
- Vite
- Axios
- ECharts / ECharts graph
- 单页应用 SPA

### 本地优先目标

先完成 localhost 可演示闭环：

1. 后端 `python run.py` 能启动。
2. 前端 `cd frontend && npm install && npm run dev` 能启动。
3. 能上传并解析至少 2 份教材。
4. 能构建知识图谱并展示节点/边。
5. 能执行跨教材整合并显示 merge/keep/remove 与压缩比。
6. 能建立 RAG 索引并返回带引用答案。
7. 能提交教师反馈并修改整合结果。
8. 能生成 `report/整合报告.md`。

### 公网部署方向

用户确认前端需要部署在公网上。当前建议：

- 先完成本地 localhost 闭环。
- 再适配魔搭创空间部署。
- 前端不要写死 `localhost`，应使用环境变量，例如：

```js
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:8000/api';
```

部署时将 `VITE_API_URL` 配置为公网后端 API 地址，或在同源部署时使用 `/api`。

### 三个并行任务方向

Agent A：知识图谱 + 跨教材整合

- 补齐 `/api/kg/build`、`/api/kg`、`/api/merge`、`/api/merge/decisions`。
- 输出 `backend/data/processed/kg_nodes.json`、`kg_edges.json`、`merged_kg.json`、`merge_decisions.json`。
- 这是当前最大缺口，优先级最高。

Agent B：RAG + 教师反馈 + 报告加固

- 兼容 `backend/data/metadata` 和 `backend/data/processed`。
- 保证 RAG、反馈、报告接口在数据不足时也有 fallback。
- 重点验证 `/api/rag/index`、`/api/rag/query`、`/api/feedback`、`/api/report/generate`。

Agent C：前端 + 文档 + 部署准备

- 修复前端可运行性，尤其是 `frontend/src/App.jsx` 中的乱码和潜在 JSX 语法错误。
- 对接真实 API，同时保留 mock fallback。
- 补齐 README、docs、部署说明、`.gitignore`。
- 适配后续魔搭创空间公网部署。

### 已修复的阻断问题

1. `backend/services/report_generator.py` 存在损坏字符串，导致后端无法 import。已修复。
2. `backend/services/llm_client.py` 的 fallback 概念抽取可能无限循环，导致 `/api/kg/build` 卡住。已改为快速有界抽取。
3. `backend/services/merger.py` 截断后没有重新计算压缩比，导致显示 `compression_ratio=1.0`。已修复为 `<0.30`。
4. `backend/services/feedback.py` 与 A 模块输出的 `merge_decisions.json` 数据结构不兼容。已兼容 list 与 dict 两种结构。
