# 前端问题修复任务派发

## 当前角色约束

当前主控不亲自做完整测试，只负责：

- 整理问题。
- 拆分任务。
- 明确文件责任边界。
- 给出可复制的 agent 提示词。
- 定义测试目标和验收标准。

测试可另派测试 agent 执行。

## 用户实测问题

1. 默认展示的数学教材很奇怪，不应该用假教材作为默认数据。`textbooks/` 目录下已有赛题给的参考书籍，应至少使用其中一本作为默认参考教材，其他教材支持手动导入。
2. 手动导入书籍后，左侧没有显示导入内容、解析状态和加载内容。
3. 中间知识图谱显示效果差：节点文字显示不完整，节点圈太小，字体过大，节点大小固定，不能体现重要程度、频率和连接关系。检索功能输入内容后页面直接不显示。
4. 生成新报告后，详情区看不到具体报告内容。

## 拆分建议

### Agent 1：教材数据流与上传显示修复

负责文件：

- `frontend/src/api.js`
- `frontend/src/App.jsx`
- `frontend/src/components/UploadPanel.jsx`
- 必要时只读后端 `backend/routers/upload.py`、`backend/routers/textbooks.py`
- 可新增本地说明文档，不要改后端核心 parser/storage

目标：

- 去掉或弱化假数学/物理 mock 默认数据。
- 前端优先展示真实 `/api/textbooks` 数据。
- 手动上传后，左侧教材列表立即刷新并显示上传/解析结果。
- 使用 `textbooks/` 下真实参考书作为本地演示建议，但不要把 PDF 提交到 Git。

### Agent 2：知识图谱视觉与交互修复

负责文件：

- `frontend/src/components/GraphView.jsx`
- `frontend/src/App.jsx` 中与节点点击详情相关的最小改动
- `frontend/src/styles/index.css`
- 不改后端

目标：

- 节点大小根据 `frequency`、degree、或连接数动态变化。
- 字体大小和节点大小匹配，避免文字溢出。
- 节点 label 显示短名，完整名放 tooltip/右侧详情。
- 边和节点关系更清楚，能看出核心节点和连接网络。
- 点击节点后右侧显示真实节点字段。

### Agent 3：RAG 检索崩溃与报告详情修复

负责文件：

- `frontend/src/components/RagPanel.jsx`
- `frontend/src/components/ReportPanel.jsx`
- `frontend/src/api.js`
- 必要时只读 `backend/routers/rag.py`、`backend/routers/report.py`

目标：

- 检索输入后页面不能白屏。
- 对 RAG 返回字段做兼容：`data.answer`、`data.citations`、`data.source_chunks` 缺失时也不崩。
- 报告生成后，详情区能显示报告 Markdown 内容。
- 若 `/api/report/generate` 只返回路径或 summary，则调用 `/api/report/latest` 获取内容。

### Agent 4：测试验收

负责：

- 只测试，不改功能代码。
- 记录每一步结果、截图、失败接口和浏览器 Console 错误。

测试顺序：

1. 后端启动。
2. 前端启动。
3. 页面首次打开不应显示奇怪假教材。
4. 上传教材后左侧列表刷新。
5. 解析教材后能看到章节/状态。
6. 构建图谱后中心图谱可读。
7. 点击节点显示详情。
8. RAG 输入问题后页面不白屏，显示答案和引用。
9. 生成报告后能看到报告正文。
